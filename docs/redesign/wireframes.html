EggSpend — Annotated Wireframes

Five redesign directions from the UX audit, each shown as Home / Transactions / Budget — the three screens with the worst measured density and hierarchy problems. Wireframes are structural (grayscale = layout, not final styling).

How to read: orange numbered dots 1 inside each phone match the notes beneath it. Every note names the audit finding it fixes. The dashed red line (Direction B) marks the fold on a 6.3″ phone. Tab bar and status bar are constant across directions.

A · Quiet Nest Minimal Apple Style

Trust the platform: native inset-grouped lists, collapsing large titles, standard toolbars. Brand survives as accent color + the egg ring, not as scenery. Hierarchy from type scale, not card chrome.

B · Glanceable Home Compact Dashboard · Recommended

The 2-second scan is the product. Home becomes a fixed dashboard that fits one screenful: Safe-to-Spend hero, Apple-Health-style stat grid, tight activity list, one insight line. Brand shrinks rather than disappears. Other tabs get density fixes only.

More options (notes, budget, transfer) ▾
  1. Amount-first, keypad already up — no tapping into a field, no keyboard-mode switch.
  2. Merchant memory read back: recent/frequent titles as one-tap chips, powered by the CategoryRuleEngine rules the app already records but never reads.
